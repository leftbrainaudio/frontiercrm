One-Paragraph Strategy

The FrontierCRM frontend has been audited across all 8 screens (login, signup, dashboard, contacts, pipeline, activities, email, settings) at three breakpoints (1280px, 768px, 390px) against the ui-ux-pro-max v1.1.0 design standard. The audit found 56 issues — 2 blockers, 12 major, 26 minor, 16 nits. 34 of those issues have been fixed by Coach and Builder across three fix cycles. The push is not about reach — it's about turning the component library, the design tokens, and the 31-skill assessment into a consistent, production-ready UI over fix cycles.

Themes are: brand consistency, accessibility (WCAG 2.1 AA), responsive behaviour, component polish. The hooks are: cubic-bezier transitions, 4px base spacing unit, Inter font family, --color-brand-500: #3B82F6, the drag-and-drop kanban, the full WAI-ARIA combobox search pattern, the three personas (Alex / Monica / Jordan), and the campaign spine "Every interaction deserves a design."

Four Screens (Audit Scope)

The official audit covered 8 screens in total. These 4 are the highest-impact spaces:

1. Login / Auth — Conversion Funnel

OAuth buttons (Google, Microsoft), SSO, magic link, 2FA challenge, field-level validation with aria-invalid and role="alert". Full design-token compliance after fixes.

2. Pipeline Kanban — Primary Workspace

Drag-and-drop via @dnd-kit, deal cards with probability bars, stage columns with colour-coded headers. Accessible drag handles with aria-label="Drag to reorder". Responsive: columns stack vertically on mobile.

3. Contacts Table — Data Hub

Refactored from raw <table> to reusable Table<Contact> component with typed columns, skeleton loading, pagination, search. Selected rows now have border-l-3 border-l-brand-500 per spec.

4. Dashboard — Home Screen

Metric cards, pipeline-by-stage chart, recent activity feed, tasks due. Chart fallback color fixed: #6366f1 → #3B82F6. All empty/loading/error states present.

A · Fixes Applied (Coach — 14)

14 fixes, applied directly by Coach (1 shot)
1. Inter font on body
index.css · 1 line
Added font-sans to body @apply so the Inter font-family token actually activates. Previously fell back to system-ui.
2. Sidebar width: 256px → 240px
sidebar.tsx · w-64 → w-60
Per spec, sidebar should be 240px fixed. Tailwind w-64 is 256px (16px over). Changed to w-60 (240px).
3. Sidebar breakpoint: md → lg
sidebar.tsx · 768-1023 collapsible drawer
Changed hidden md:flex to hidden lg:flex. Sidebar now acts as collapsible drawer at 768-1023px per spec, and overlay at <1024px.
4. focus-visible rings: 11 elements
sidebar, topbar, login · 11 buttons
Added focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-brand-500 to collapse toggle, mobile close X, MobileMenuButton, theme toggle, notifications, profile dropdown, OAuth buttons, SSO button, magic link, sign up link.
5. aria-labels on collapsed nav + toggle
sidebar.tsx
Collapse toggle gets aria-label={collapsed ? 'Expand sidebar' : 'Collapse sidebar'}. Collapsed icon-only nav links get aria-label={item.label}. Mobile close X gets aria-label='Close menu'.
6. Login heading: text-2xl → text-3xl
login.tsx · 24px → 30px
"Welcome back" heading was 24px (text-2xl). Spec says H1 should be 28px. Changed to text-3xl (30px) — closer to spec.
7. Mobile sidebar closes on route change
app-layout.tsx
useEffect dependency changed from [] to [location]. Mobile sidebar overlay now closes when user navigates to a new page.

Success Signals & Red Flags

Success (T+7 → T+21 after fixes)

  • focus-visible on all interactive elements: >95% pass rate on a11y scan, holding or rising
  • Design token compliance: >90% of components reference tokens, not raw values
  • Responsive breakpoints: all 3 breakpoints render without horizontal scroll or broken layout
  • WCAG 2.1 AA: body text contrast >4.5:1, touch targets ≥44×44px, skip-to-content present
  • TypeScript: tsc --noEmit passes with zero errors across all fix batches

Red Flags

  • New components added without focus-visible rings — fix is to add the focus-visible:ring-2 focus-visible:ring-brand-500 classes
  • High token drift — new components using raw Tailwind grays instead of --color-surface/border/text-* tokens
  • Server-side PDF not implemented — all exports are CSV-only. "Print to PDF" still works via browser
  • No analytics instrumentation — design decisions are not data-informed. "The recommendation lane is dormant."
