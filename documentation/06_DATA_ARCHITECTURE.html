    FrontierCRM — Data Model Schema

    Complete database schema for FrontierCRM Stack: PostgreSQL 16 + TimescaleDB + Redis + Meilisearch + Cloudflare R2 Tenancy: Row-Level Security (RLS) via tenant_id on every table Target: SMB/Mid-Market (10–200 employees) Dated: 2026-06-28

    1. Schema Overview

    Naming Conventions

    Convention Rule
    Table names snake_case, plural (e.g., contacts, deal_stages)
    Primary key id UUID DEFAULT gen_random_uuid() on every table
    Tenant column tenant_id UUID NOT NULL on every tenant-scoped table
    Timestamps created_at, updated_at, deleted_at (soft delete via nullable)
    Foreign keys table_name_id (e.g., contact_id, deal_id)
    Indexes ix_table_column for single, ix_table_col1_col2 for composite

    Schema Definition

    All tables live under the public schema (single schema for simplicity — no schema-per-tenant). TimescaleDB hypertables are created via SELECT create_hypertable().

    2. Multi-Tenancy Implementation

    RLS Policy

    Every tenant-scoped table has: - tenant_id UUID NOT NULL DEFAULT current_setting('app.current_tenant_id')::UUID - An RLS policy: tenant_id = current_setting('app.current_tenant_id')::UUID

    Middleware Flow

    Incoming Request
  → Clerk JWT verified (FastAPI middleware)
  → Extract org_id from JWT claims
  → Execute: SET SESSION app.current_tenant_id = '{org_id}'
  → Execute: SET SESSION app.current_user_id = '{user_id}'
  → All subsequent queries in this session apply RLS automatically

    RLS Policy Template

    -- Applied to EVERY tenant-scoped table
CREATE POLICY tenant_isolation ON {table_name}
  FOR ALL
  USING (tenant_id = current_setting('app.current_tenant_id')::UUID)
  WITH CHECK (tenant_id = current_setting('app.current_tenant_id')::UUID);

-- Force RLS
ALTER TABLE {table_name} ENABLE ROW LEVEL SECURITY;
ALTER TABLE {table_name} FORCE ROW LEVEL SECURITY;

    What is NOT tenant-scoped

    • roles — global role definitions (Admin, Manager, Rep, Read-only)
    • permissions — global permission definitions
    • migrations — Alembic schema versioning

    Everything else carries tenant_id.

    3. Core Business Models

    3.1 Contact

    CREATE TABLE contacts (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL,

    -- Identity
    first_name VARCHAR(128) NOT NULL,
    last_name VARCHAR(128) NOT NULL,
    salutation VARCHAR(32),                    -- Mr., Ms., Dr., etc.
    job_title VARCHAR(256),
    company VARCHAR(256),

    -- Communication
    emails JSONB NOT NULL DEFAULT '[]'::jsonb,  -- [{type: 'work', email: 'a@b.com', primary: true}]
    phones JSONB NOT NULL DEFAULT '[]'::jsonb,  -- [{type: 'mobile', number: '+15551234', primary: true}]
    social_profiles JSONB DEFAULT '{}'::jsonb,  -- {linkedin: url, twitter: handle}

    -- Address
    address_line1 VARCHAR(256),
    address_line2 VARCHAR(256),
    city VARCHAR(128),
    state VARCHAR(64),
    postal_code VARCHAR(32),
    country VARCHAR(64),

    -- Classification
    lifecycle_stage VARCHAR(32) DEFAULT 'lead'
        CHECK (lifecycle_stage IN ('lead', 'prospect', 'active', 'churned', 'inactive')),
    source VARCHAR(64),                         -- Referral, Website, Cold Call, Partner
    owner_id UUID REFERENCES users(id),

    -- Relationship
    account_id UUID REFERENCES accounts(id) ON DELETE SET NULL,

    -- Metadata
    tags TEXT[],                                -- PostgreSQL array of tags
    custom_fields JSONB DEFAULT '{}'::jsonb,    -- {field_def_id: value}
    description TEXT,
    last_contacted_at TIMESTAMPTZ,
    last_activity_at TIMESTAMPTZ,

    -- System
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    deleted_at TIMESTAMPTZ,                    -- Soft delete

    -- Constraints
    CONSTRAINT valid_email CHECK (jsonb_typeof(emails) = 'array'),
    CONSTRAINT valid_phone CHECK (jsonb_typeof(phones) = 'array')
);

-- Indexes
CREATE INDEX ix_contacts_tenant_id ON contacts(tenant_id);
CREATE INDEX ix_contacts_owner_id ON contacts(owner_id);
CREATE INDEX ix_contacts_account_id ON contacts(account_id);
CREATE INDEX ix_contacts_name ON contacts(last_name, first_name);
CREATE INDEX ix_contacts_lifecycle_stage ON contacts(tenant_id, lifecycle_stage);
CREATE INDEX ix_contacts_tags ON contacts USING GIN(tags);
CREATE INDEX ix_contacts_custom_fields ON contacts USING GIN(custom_fields jsonb_path_ops);
CREATE INDEX ix_contacts_deleted_at ON contacts(deleted_at) WHERE deleted_at IS NOT NULL;

-- Full-text search vector (fallback when Meilisearch is unavailable)
ALTER TABLE contacts ADD COLUMN search_vector tsvector
    GENERATED ALWAYS AS (
        to_tsvector('english', coalesce(first_name, '') || ' ' || coalesce(last_name, '') || ' ' || coalesce(company, '') || ' ' || coalesce(description, ''))
    ) STORED;
CREATE INDEX ix_contacts_search ON contacts USING GIN(search_vector);

    Notes: - emails and phones are JSONB arrays (not separate tables). Contact-level sync from Gmail/Outlook works with the primary email address; secondary emails are stored but CRM operations use the primary: true entry. - custom_fields is JSONB with field_definition_id as keys. The CustomField system (Section 8) defines the schema per tenant. - Soft delete: deleted_at IS NOT NULL means deleted. RLS still applies to deleted rows.

    3.2 Account

    CREATE TABLE accounts (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL,

    -- Identity
    name VARCHAR(256) NOT NULL,
    domain VARCHAR(256),                       -- Company domain (unique per tenant)
    industry VARCHAR(128),
    account_type VARCHAR(32) DEFAULT 'customer'
        CHECK (account_type IN ('customer', 'partner', 'vendor', 'competitor', 'other')),
    account_tier VARCHAR(32) DEFAULT 'standard'
        CHECK (account_tier IN ('enterprise', 'standard', 'basic')),

    -- Financial
    annual_revenue NUMERIC(14, 2),
    employee_count INTEGER,
    currency VARCHAR(3) DEFAULT 'USD',

    -- Address
    billing_address JSONB DEFAULT '{}'::jsonb,
    shipping_address JSONB DEFAULT '{}'::jsonb,
    phone VARCHAR(64),

    -- CRM
    owner_id UUID REFERENCES users(id),
    website VARCHAR(512),
    description TEXT,
    tags TEXT[],

    -- Relationship
    parent_account_id UUID REFERENCES accounts(id) ON DELETE SET NULL,

    -- Metadata
    custom_fields JSONB DEFAULT '{}'::jsonb,
    last_activity_at TIMESTAMPTZ,

    -- System
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    deleted_at TIMESTAMPTZ,

    CONSTRAINT uq_account_domain UNIQUE (tenant_id, domain)
);

CREATE INDEX ix_accounts_tenant_id ON accounts(tenant_id);
CREATE INDEX ix_accounts_owner_id ON accounts(owner_id);
CREATE INDEX ix_accounts_name ON accounts(tenant_id, name);
CREATE INDEX ix_accounts_type ON accounts(tenant_id, account_type);
CREATE INDEX ix_accounts_parent ON accounts(parent_account_id);
CREATE INDEX ix_accounts_tags ON accounts USING GIN(tags);
CREATE INDEX ix_accounts_deleted_at ON accounts(deleted_at) WHERE deleted_at IS NOT NULL;

    3.3 User

    CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL,

    -- Identity (synced from Clerk via webhook)
    clerk_user_id VARCHAR(256) NOT NULL,       -- Clerk's user ID
    email VARCHAR(256) NOT NULL,
    first_name VARCHAR(128),
    last_name VARCHAR(128),
    avatar_url VARCHAR(512),

    -- CRM
    title VARCHAR(256),
    phone VARCHAR(64),
    timezone VARCHAR(64) DEFAULT 'UTC',
    locale VARCHAR(10) DEFAULT 'en-US',
    settings JSONB DEFAULT '{}'::jsonb,        -- User preferences (notification prefs, default view, etc.)

    -- Status
    is_active BOOLEAN DEFAULT true,
    last_seen_at TIMESTAMPTZ,

    -- System
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    deleted_at TIMESTAMPTZ,

    CONSTRAINT uq_user_clerk UNIQUE (clerk_user_id),
    CONSTRAINT uq_user_email_per_tenant UNIQUE (tenant_id, email)
);

CREATE INDEX ix_users_tenant_id ON users(tenant_id);
CREATE INDEX ix_users_active ON users(tenant_id, is_active) WHERE is_active = true;
CREATE INDEX ix_users_deleted_at ON users(deleted_at) WHERE deleted_at IS NOT NULL;

    3.4 Team

    CREATE TABLE teams (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL,

    name VARCHAR(256) NOT NULL,
    description TEXT,
    parent_team_id UUID REFERENCES teams(id) ON DELETE SET NULL,
    is_default BOOLEAN DEFAULT false,

    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),

    CONSTRAINT uq_team_name UNIQUE (tenant_id, name)
);

CREATE INDEX ix_teams_tenant_id ON teams(tenant_id);

-- Team membership (many-to-many)
CREATE TABLE team_members (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL,
    team_id UUID NOT NULL REFERENCES teams(id) ON DELETE CASCADE,
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    role VARCHAR(32) DEFAULT 'member'
        CHECK (role IN ('admin', 'manager', 'member')),

    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),

    CONSTRAINT uq_team_member UNIQUE (team_id, user_id)
);

CREATE INDEX ix_team_members_team ON team_members(team_id);
CREATE INDEX ix_team_members_user ON team_members(user_id);

    4. Activity & Communication Models

    4.1 Activity (Polymorphic Base)

    The activities table is the unified activity timeline — every noteworthy event in the CRM gets an entry here. It's a TimescaleDB hypertable for efficient time-range queries.

    CREATE TABLE activities (
    id UUID NOT NULL DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL,
    user_id UUID REFERENCES users(id) ON DELETE SET NULL,

    -- Polymorphic source
    source_type VARCHAR(32) NOT NULL,          -- 'note', 'task', 'email', 'event', 'call', 'system', 'deal_update'
    source_id UUID NOT NULL,                   -- FK to the specific entity (polymorphic via source_type)

    -- Context
    contact_id UUID REFERENCES contacts(id) ON DELETE SET NULL,
    deal_id UUID REFERENCES deals(id) ON DELETE SET NULL,
    account_id UUID REFERENCES accounts(id) ON DELETE SET NULL,
    team_id UUID REFERENCES teams(id) ON DELETE SET NULL,

    -- Activity data
    action VARCHAR(64) NOT NULL,               -- 'created', 'updated', 'called', 'emailed', 'note_added', 'stage_changed', etc.
    description TEXT,
    metadata JSONB DEFAULT '{}'::jsonb,        -- Extra context (old_value, new_value, duration, etc.)

    -- Temporal
    occurred_at TIMESTAMPTZ NOT NULL,          -- When the activity actually happened (vs when we recorded it)
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Convert to hypertable (must be done outside CREATE TABLE)
-- SELECT create_hypertable('activities', 'occurred_at', chunk_time_interval => INTERVAL '7 days');

CREATE INDEX ix_activities_tenant ON activities(tenant_id, occurred_at DESC);
CREATE INDEX ix_activities_contact ON activities(tenant_id, contact_id, occurred_at DESC);
CREATE INDEX ix_activities_deal ON activities(tenant_id, deal_id, occurred_at DESC);
CREATE INDEX ix_activities_user ON activities(tenant_id, user_id, occurred_at DESC);
CREATE INDEX ix_activities_source ON activities(source_type, source_id);
CREATE INDEX ix_activities_action ON activities(tenant_id, action);

    4.2 Note

    CREATE TABLE notes (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL,
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,

    title VARCHAR(512),
    body TEXT NOT NULL,                        -- Markdown content
    body_html TEXT,                            -- Rendered HTML (cached)

    -- Linking (at least one should be set)
    contact_id UUID REFERENCES contacts(id) ON DELETE CASCADE,
    deal_id UUID REFERENCES deals(id) ON DELETE CASCADE,
    account_id UUID REFERENCES accounts(id) ON DELETE CASCADE,

    -- Classification
    is_pinned BOOLEAN DEFAULT false,
    visibility VARCHAR(16) DEFAULT 'public'
        CHECK (visibility IN ('public', 'team', 'private')),

    -- System
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    deleted_at TIMESTAMPTZ
);

CREATE INDEX ix_notes_tenant_id ON notes(tenant_id);
CREATE INDEX ix_notes_user_id ON notes(user_id);
CREATE INDEX ix_notes_contact ON notes(contact_id);
CREATE INDEX ix_notes_deal ON notes(deal_id);
CREATE INDEX ix_notes_account ON notes(account_id);
CREATE INDEX ix_notes_visibility ON notes(tenant_id, visibility);

    4.3 Task

    CREATE TABLE tasks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL,
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    assignee_id UUID REFERENCES users(id) ON DELETE SET NULL,

    title VARCHAR(512) NOT NULL,
    description TEXT,

    -- Status
    status VARCHAR(32) NOT NULL DEFAULT 'pending'
        CHECK (status IN ('pending', 'in_progress', 'completed', 'cancelled')),
    priority VARCHAR(16) NOT NULL DEFAULT 'medium'
        CHECK (priority IN ('low', 'medium', 'high', 'urgent')),

    -- Dates
    due_at TIMESTAMPTZ,
    completed_at TIMESTAMPTZ,
    reminder_at TIMESTAMPTZ,

    -- Linking
    contact_id UUID REFERENCES contacts(id) ON DELETE CASCADE,
    deal_id UUID REFERENCES deals(id) ON DELETE CASCADE,
    account_id UUID REFERENCES accounts(id) ON DELETE CASCADE,

    -- Recurrence
    recurrence_rule VARCHAR(256),               -- RRULE string (RFC 5545), NULL = non-recurring
    recurrence_end_at TIMESTAMPTZ,

    -- System
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    deleted_at TIMESTAMPTZ
);

CREATE INDEX ix_tasks_tenant_id ON tasks(tenant_id);
CREATE INDEX ix_tasks_assignee ON tasks(tenant_id, assignee_id);
CREATE INDEX ix_tasks_status ON tasks(tenant_id, status);
CREATE INDEX ix_tasks_due ON tasks(tenant_id, due_at) WHERE due_at IS NOT NULL AND deleted_at IS NULL;
CREATE INDEX ix_tasks_contact ON tasks(contact_id);
CREATE INDEX ix_tasks_deal ON tasks(deal_id);

    4.4 Email

    CREATE TABLE emails (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL,
    user_id UUID REFERENCES users(id) ON DELETE SET NULL,  -- CRM user who owns/synced this email

    -- Email identity
    message_id VARCHAR(512),                   -- Internet Message-ID (RFC 2822)
    thread_id VARCHAR(256),                    -- CRM thread grouping ID
    external_id VARCHAR(512),                  -- Provider's message ID (Gmail ID, Graph ID)
    external_thread_id VARCHAR(512),           -- Provider's thread ID

    -- Headers
    subject TEXT,
    from_address VARCHAR(256) NOT NULL,
    from_name VARCHAR(256),
    to_addresses TEXT[] NOT NULL,
    cc_addresses TEXT[],
    bcc_addresses TEXT[],
    reply_to VARCHAR(256),

    -- Body
    body_text TEXT,                            -- Plain text body
    body_html TEXT,                            -- HTML body
    snippet VARCHAR(512),                      -- Short preview

    -- Status
    direction VARCHAR(8) NOT NULL              -- 'inbound', 'outbound'
        CHECK (direction IN ('inbound', 'outbound')),
    is_read BOOLEAN DEFAULT false,
    is_starred BOOLEAN DEFAULT false,
    labels TEXT[],                             -- Gmail labels / Outlook categories

    -- Dates
    sent_at TIMESTAMPTZ NOT NULL,              -- Original send date from provider
    received_at TIMESTAMPTZ NOT NULL,          -- When CRM received/synced it

    -- Linking (CRM context)
    contact_id UUID REFERENCES contacts(id) ON DELETE SET NULL,
    deal_id UUID REFERENCES deals(id) ON DELETE SET NULL,
    account_id UUID REFERENCES accounts(id) ON DELETE SET NULL,

    -- System
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    deleted_at TIMESTAMPTZ
);

CREATE INDEX ix_emails_tenant_id ON emails(tenant_id);
CREATE INDEX ix_emails_user_id ON emails(user_id);
CREATE INDEX ix_emails_thread ON emails(tenant_id, thread_id);
CREATE INDEX ix_emails_contact ON emails(tenant_id, contact_id, sent_at DESC);
CREATE INDEX ix_emails_deal ON emails(tenant_id, deal_id, sent_at DESC);
CREATE INDEX ix_emails_direction ON emails(tenant_id, direction, sent_at DESC);
CREATE INDEX ix_emails_sent_at ON emails(tenant_id, sent_at DESC);
CREATE INDEX ix_emails_from ON emails(from_address);
CREATE INDEX ix_emails_external_id ON emails(tenant_id, external_id) WHERE external_id IS NOT NULL;
CREATE INDEX ix_emails_thread_external ON emails(tenant_id, external_thread_id) WHERE external_thread_id IS NOT NULL;

    4.5 Event (Calendar)

    CREATE TABLE events (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL,
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,

    -- Event identity
    external_id VARCHAR(512),                  -- Provider event ID (Google event ID, Graph event ID)
    external_calendar_id VARCHAR(256),         -- Which calendar this belongs to
    ical_uid VARCHAR(256),                     -- iCal UID for CalDAV sync

    -- Core fields
    title VARCHAR(512) NOT NULL,
    description TEXT,
    location VARCHAR(512),

    -- Timing
    start_at TIMESTAMPTZ NOT NULL,
    end_at TIMESTAMPTZ NOT NULL,
    is_all_day BOOLEAN DEFAULT false,
    timezone VARCHAR(64) DEFAULT 'UTC',

    -- Recurrence
    recurrence_rule VARCHAR(4096),             -- RRULE string(s), NULL = non-recurring
    recurrence_exception_dates TIMESTAMPTZ[],  -- EXDATE values
    is_recurrence_instance BOOLEAN DEFAULT false,
    recurring_event_id UUID,                   -- FK to the parent recurring event

    -- Attendees
    attendees JSONB DEFAULT '[]'::jsonb,       -- [{email, name, status: 'accepted'|'tentative'|'declined'|'needs_action'}]
    organizer JSONB DEFAULT '{}'::jsonb,       -- {email, name, self: boolean}
    is_organizer BOOLEAN DEFAULT false,

    -- Status
    status VARCHAR(16) DEFAULT 'confirmed'
        CHECK (status IN ('confirmed', 'tentative', 'cancelled')),
    visibility VARCHAR(16) DEFAULT 'default'
        CHECK (visibility IN ('default', 'public', 'private', 'confidential')),

    -- Meet / video conference
    conference_link VARCHAR(512),
    conference_data JSONB DEFAULT '{}'::jsonb, -- Provider-specific conferencing info

    -- Linking (CRM context)
    contact_id UUID REFERENCES contacts(id) ON DELETE SET NULL,
    deal_id UUID REFERENCES deals(id) ON DELETE SET NULL,
    account_id UUID REFERENCES accounts(id) ON DELETE SET NULL,

    -- Sync tracking
    sync_provider VARCHAR(32),                 -- 'google_calendar', 'outlook_calendar', 'caldav'
    etag VARCHAR(256),                         -- For CalDAV sync

    -- System
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    deleted_at TIMESTAMPTZ,

    CONSTRAINT uq_external_event UNIQUE (tenant_id, sync_provider, external_id) WHERE external_id IS NOT NULL
);

CREATE INDEX ix_events_tenant_id ON events(tenant_id);
CREATE INDEX ix_events_user_id ON events(user_id);
CREATE INDEX ix_events_time ON events(tenant_id, start_at, end_at);
CREATE INDEX ix_events_contact ON events(tenant_id, contact_id) WHERE contact_id IS NOT NULL;
CREATE INDEX ix_events_deal ON events(tenant_id, deal_id) WHERE deal_id IS NOT NULL;
CREATE INDEX ix_events_provider ON events(tenant_id, sync_provider);

    5. Sales Pipeline Models

    5.1 Pipeline

    CREATE TABLE pipelines (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL,

    name VARCHAR(256) NOT NULL,
    description TEXT,
    is_default BOOLEAN DEFAULT false,
    deal_forecast_category VARCHAR(32) DEFAULT 'revenue' CHECK (deal_forecast_category IN ('revenue', 'volume', 'both')),

    -- Display settings
    color VARCHAR(7) DEFAULT '#3B82F6',

    -- System
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),

    CONSTRAINT uq_pipeline_name UNIQUE (tenant_id, name)
);

CREATE INDEX ix_pipelines_tenant ON pipelines(tenant_id);

    5.2 Stage

    CREATE TABLE stages (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL,
    pipeline_id UUID NOT NULL REFERENCES pipelines(id) ON DELETE CASCADE,

    name VARCHAR(256) NOT NULL,
    display_order INTEGER NOT NULL DEFAULT 0,

    -- Configuration

    8. Custom Fields System

    The custom fields system allows tenants to define their own schema on top of Contacts, Deals, and Accounts without schema migrations.

    8.1 Field Definition

    CREATE TABLE custom_field_definitions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL,

    -- Identity
    name VARCHAR(128) NOT NULL,                -- Display name (e.g., "Lead Source")
    key VARCHAR(128) NOT NULL,                 -- JSONB key (e.g., "lead_source" — snake_case, auto-generated)
    description TEXT,
    entity VARCHAR(32) NOT NULL                -- 'contact', 'deal', 'account', 'event', 'task'
        CHECK (entity IN ('contact', 'deal', 'account', 'event', 'task')),

    -- Field type
    field_type VARCHAR(32) NOT NULL
        CHECK (field_type IN (
            'text', 'textarea', 'number', 'decimal', 'boolean',
            'date', 'datetime', 'select', 'multi_select',
            'url', 'email', 'phone', 'currency',
            'user', 'contact_link', 'deal_link'
        )),

    -- Select options (for select/multi_select types)
    options JSONB DEFAULT '[]'::jsonb,         -- [{value: 'opt1', label: 'Option 1', color: '#ff0000'}]

    -- Validation
    is_required BOOLEAN DEFAULT false,
    validation_rules JSONB DEFAULT '{}'::jsonb, -- {min: 0, max: 100, regex: '...'}
    default_value JSONB,

    -- Display
    display_order INTEGER DEFAULT 0,
    section VARCHAR(128),                      -- UI grouping (e.g., "Sales Info", "Technical Details")

    -- System
    is_active BOOLEAN DEFAULT true,
    is_system BOOLEAN DEFAULT false,           -- Predefined by FrontierCRM (e.g., "Deal Value")
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),

    CONSTRAINT uq_custom_field_key UNIQUE (tenant_id, entity, key)
);

CREATE INDEX ix_custom_field_defs_entity ON custom_field_definitions(tenant_id, entity, display_order);

    How it works: - Each entity has a custom_fields JSONB column. - key in the definition maps to the JSONB key in custom_fields. - Validation is enforced at the FastAPI/Pydantic layer, not in the DB (performance). - The frontend renders the appropriate widget based on field_type. - System fields (is_system = true) are created by migrations and cannot be deleted by tenants.

    10.4 Pipeline Snapshot (Materialized View)

    -- Materialized view refreshed nightly (or on-demand via Celery)
CREATE MATERIALIZED VIEW pipeline_snapshots AS
SELECT
    d.tenant_id,
    d.id AS deal_id,
    d.pipeline_id,
    d.stage_id,
    s.name AS stage_name,
    s.display_order AS stage_order,
    d.owner_id,
    d.value,
    d.probability,
    d.value * (d.probability / 100.0) AS weighted_value,
    d.expected_close_at,
    d.actual_close_at,
    d.created_at AS deal_created_at,
    d.stage_entered_at,
    CASE
        WHEN d.won_at IS NOT NULL THEN 'won'
        WHEN d.lost_at IS NOT NULL THEN 'lost'
        ELSE 'open'
    END AS deal_status,
    u.first_name || ' ' || u.last_name AS owner_name,
    c.first_name || ' ' || c.last_name AS contact_name,
    a.name AS account_name
FROM deals d
JOIN stages s ON d.stage_id = s.id
LEFT JOIN users u ON d.owner_id = u.id
LEFT JOIN contacts c ON d.primary_contact_id = c.id
LEFT JOIN accounts a ON d.account_id = a.id
WHERE d.deleted_at IS NULL;

CREATE UNIQUE INDEX ix_pipeline_snapshots_deal ON pipeline_snapshots(tenant_id, deal_id);
CREATE INDEX ix_pipeline_snapshots_stage ON pipeline_snapshots(tenant_id, stage_id);
CREATE INDEX ix_pipeline_snapshots_owner ON pipeline_snapshots(tenant_id, owner_id);
CREATE INDEX ix_pipeline_snapshots_status ON pipeline_snapshots(tenant_id, deal_status);

    tenant_id UUID NOT NULL,

    refresh_type VARCHAR(32) NOT NULL,         -- 'pipeline_snapshot', 'user_metrics', 'report_cache'
    entity_id UUID,                            -- Optional: specific entity to refresh
    priority INTEGER DEFAULT 0,

    status VARCHAR(16) DEFAULT 'pending'
        CHECK (status IN ('pending', 'processing', 'completed', 'failed')),
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    processed_at TIMESTAMPTZ
);

CREATE INDEX ix_analytics_refresh_status ON analytics_refresh_queue(tenant_id, status, priority DESC);

    11. Index Strategy

    Performance Principles

    1. Every tenant query starts with tenant_id — all non-unique indexes are prefixed with tenant_id to support RLS filtering first.
    2. Covering indexes for hot paths — the pipeline kanban board query (SELECT deals WHERE pipeline_id = X AND tenant_id = Y ORDER BY stage_order, updated_at DESC) is covered by ix_deals_stage.
    3. JSONB GIN indexes — use jsonb_path_ops for faster equality/containment queries on custom fields.
    4. Partial indexes for filtered queries — WHERE deleted_at IS NULL on active records, WHERE status = 'active' on sync connections.
    5. TimescaleDB hyperindexes — hypertables automatically create space-partitioned indexes; we add tenant- and entity-specific ones.

    Index Priority Matrix

    Index Purpose Expected QPS Type
    ix_deals_stage Pipeline board display High B-tree composite
    ix_activities_contact Contact timeline High B-tree with time ordering
    ix_activities_deal Deal timeline High B-tree with time ordering
    ix_contacts_name Autocomplete search High B-tree
    ix_emails_thread Email thread view Medium B-tree
    ix_events_time Calendar view Medium B-tree range
    ix_sync_states_next Sync scheduler Medium B-tree partial
    ix_deals_custom_fields Custom field filter Low (per tenant) GIN jsonb_path_ops

    Full-Text Search Indexes (Fallback)

    PostgreSQL search_vector columns (GIN-indexed) exist on contacts, deals, and notes as a fallback when Meilisearch is unavailable. These are NOT the primary search path — they protect against Meilisearch downtime (eventual consistency means there's a brief window where Postgres has the source of truth that Meilisearch hasn't indexed yet).

    12. Migration Strategy

    Tool: Alembic + SQLAlchemy 2.0 async

    # Example migration pattern
"""add contact search vector

Revision ID: abc123
Revises: def456
Create Date: 2026-07-01 10:00:00
"""
from alembic import op
import sqlalchemy as sa
from sqlalchemy.dialects import postgresql

def upgrade():
    op.add_column('contacts',
        sa.Column('search_vector', postgresql.TSVECTOR,
                  sa.Computed("to_tsvector('english', ...)", persisted=True))
    )
    op.create_index('ix_contacts_search', 'contacts', ['search_vector'],
                    postgresql_using='gin')

def downgrade():
    op.drop_index('ix_contacts_search')
    op.drop_column('contacts', 'search_vector')

    Migration Principles

    1. Always backwards-compatible — no column drops for 2 weeks after deprecation.
    2. Zero-downtime deploys — use ADD COLUMN ... DEFAULT NULL (Postgres 11+ handles this without table rewrite) for new columns. Backfill with Celery tasks.
    3. Large table migrations — use pt-online-schema-change equivalent for tables >10M rows (unlikely at SMB scale).
    4. Hypertable migration — TimescaleDB tables cannot be migrated normally. Schema changes happen via separate hypertable commands.
    5. Seeds — system roles, default pipelines/stages, and standard permissions are seeded in migration 001_base.py.
