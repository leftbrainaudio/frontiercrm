    Development Roadmap & Release Plan (v2)

    Status: ✅ Updated to match current stack and priorities. Stack: Django 5 + DRF + React 19 + Tailwind CSS v4 + Supabase (PostgreSQL) See: 11_DECISION_LOG.md for all architecture decisions.

    Build Philosophy

    1. Design system first — No more divergence between specs and implementation. Fix the design system, then build on top of it.
    2. Supabase for dev — Free PostgreSQL during development. Migrate from SQLite.
    3. Organized components — Atomic design hierarchy (atoms → molecules → organisms → templates). Storybook for documentation.
    4. Agile DevOps — 1-week sprints via kanban, CI/CD, code review gating.
    5. API-first — All features built as APIs first, UI second.

    Current Tech Stack

    Django 5 🐍 + DRF + SimpleJWT + React 19 ⚛️ + Tailwind CSS v4
+ Vite + react-router v7 + Zustand + TanStack Query + Lucide Icons
+ Supabase 🗄️ (PostgreSQL free tier) + psycopg[binary]

    UI Framework Architecture Decision

    Chosen path: Build a Chakra-like component library on top of Tailwind v4 (not migrate to Chakra).

    Rationale: 1. No rewrite needed — 15+ existing custom components would need to be replaced under Chakra 2. Tailwind v4 is already CSS-first — @theme block in index.css IS our design token system 3. Faster to production — fix tokens → organize → document → build missing pieces vs. rewrite everything 4. Same end result — organized, documented, accessible, design-token-driven components 5. Add missing teeth — @radix-ui/react-* for accessibility primitives (what Chakra uses internally), Storybook for documentation

    Stack additions: - @fontsource/inter + @fontsource/jetbrains-mono — typography - @storybook/react + @storybook/addon-themes — component documentation - @radix-ui/react-dialog — modal foundation (already partially done) - @radix-ui/react-select — accessible select - @radix-ui/react-dropdown-menu — accessible dropdowns

    Supabase Plan

    1. Sign up at supabase.com (free tier)
    2. Create project → get connection string
    3. Set DATABASE_URL=postgresql://postgres:[password]@[host]:[port]/postgres in .env
    4. Run python manage.py migrate
    5. Run seed command to restore demo data
    6. Document in README so it survives rebuilds
