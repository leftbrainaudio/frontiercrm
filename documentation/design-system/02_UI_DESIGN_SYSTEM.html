    1.2 Content Area Panel System

    The content area supports three panel configurations:

    Single Panel (default)      Split Panel              Detail Panel
┌────────────────────┐   ┌────────┬──────────┐   ┌────────┬──────────────┐
│                    │   │        │          │   │ List   │ Detail View  │

    • Split panel: Used for contact list→detail, deal list→preview pattern
    • Detail panel: Used for email thread viewer, activity feed alongside a record
    • Panel resizing: Drag handle between panels (min 30%/max 70% for each)
    • Panel must work at ≥768px; below that, stack vertically or use push navigation

    2. Core Components

    2.1 Button

    Variant CSS Class Background Text Hover Active Disabled
    Primary btn-primary primary-500 white primary-600 primary-700 opacity-50, cursor-not-allowed
    Secondary btn-secondary white neutral-700 neutral-100 bg neutral-200 bg opacity-40
    Outline btn-outline transparent primary-600 primary-50 bg primary-100 bg opacity-40
    Ghost btn-ghost transparent neutral-600 neutral-100 bg neutral-200 bg opacity-40
    Danger btn-danger error white error-600 error-700 opacity-50
    Link btn-link transparent primary-600 underline primary-700 opacity-40

    Sizes:

    Size Height Padding Font Size Icon Size
    xs 24px 6px 8px 12px 12px
    sm 32px 8px 12px 13px 14px
    md 40px 10px 16px 14px 16px
    lg 48px 12px 24px 16px 18px
    xl 56px 16px 32px 18px 20px

    States (all variants):

    • Default: As spec'd above
    • Hover: Background shifts one step, cursor:pointer
    • Active/Pressed: Background shifts two steps, scale(0.97) for 100ms
    • Focus: 3px ring in primary-300/primary-500, offset 2px white (WCAG focus indicator)
    • Loading: Replace label text with spinner, maintain width to prevent layout shift. Disabled while loading.
    • Disabled: Opacity reduction, cursor-not-allowed, no hover effect

    Button groups: Adjacent buttons join with 0 gap. Middle buttons have square corners. First button has left-radius, last button has right-radius.

    Dismiss behavior:

    Trigger Action
    Click X Close (no save)
    Press Escape Close (no save)
    Click overlay Close (no save) — except for destructive content
    Submit primary Close + apply

    Focus trap: Tab cycles through modal elements only. First focusable element receives focus on open. Return focus to trigger on close.

    Form elements:

    Element Component Used Notes

    Date picker Input type=date or custom popover Same 40px height
    Multi-select Chips in input area, dropdown below Same height expands
    File upload Drag-and-drop zone (dashed border, 120px height)

    5. Component States Reference

    Every interactive component supports these states. This section defines the standard implementations.

    5.1 Loading States

    Component Loading Treatment
    Button Replace text with spinner (16px, animate-spin), maintain width
    Table Skeleton rows: 3-5 placeholder rows with animated pulse gradient
    Card Skeleton: Image placeholder (40% width) + 2 text lines
    Modal Skeleton inside modal body
    Dropdown Spinner inside dropdown (centered, 24px)
    Chart Skeleton chart outline + shimmer animation
    Kanban Skeleton cards per column (2 per column, pulse animation)
    Activity feed Skeleton items: dot + line + line (3 items)
    Dashboard widget Skeleton: title bar + content box (pulse)

    Skeleton animation specs: - Base gradient: neutral-200 → neutral-100 → neutral-200 (light mode) - Dark gradient: neutral-800 → neutral-700 → neutral-800 - Animation: shimmer — translateX(-100% → 100%) over 1.5s, infinite - Border radius: match component (6px cards, 4px lines)

    5.2 Empty States

    Every data-display component with zero data must show an empty state (never a blank/empty container).

    Component Empty State Content CTA
    Pipeline Kanban "No deals yet. Ready to build your pipeline?" [+ Add Deal]
    Contact list "No contacts in this list." [Import from CSV] [Add Manually]
    Activity feed "No recent activity for this record." [Add Note]
    Dashboard widget "No data yet for this metric." [Configure Source]
    Table "No matching records found." [Clear Filters] [Create New]
    Search results "No results for '[query]'" [Clear Search]
    Timeline "No activity logged for this deal." [Log Call] [Send Email]
    Notification panel "You're all caught up!" with checkmark icon —

    Empty state visual: - Centered in the container - Large illustration (64-120px, brand color, semi-transparent) - Body text (16px, neutral-600, max-width 360px) - Primary CTA button below text - Subtle secondary action as text link below button

    5.3 Error States

    Component Error Treatment
    Form field Error: red border + error text below. Success: green border + check mark
    Page section Error banner with message + [Retry] button
    Data fetch Full error state: icon + "Could not load [name]" + [Retry]
    Toast Error toast: red left border, icon, message, persistent (no auto-dismiss)
    Whole page Error page: icon + message + [Go to Dashboard] or [Reload]

    5.4 Transition States

    State Visual Duration
    Loading → Content Fade in (opacity 0→1) 300ms
    Loading → Error Slide down error banner from top 200ms
    Empty → Loaded Items fade in staggered (each 50ms apart) 200ms per item
    Content → Loading Content fades to 0.3 opacity, skeleton appears 200ms
    Drag start Scale(1) → scale(1.05), shadow-sm → shadow-md 100ms
    Drag end Scale(1.05) → scale(1), shadow-md → shadow-sm 150ms spring

    6. Accessibility (WCAG 2.1 AA)

    6.1 Color and Contrast

    • All text meets 4.5:1 minimum contrast ratio (body text) and 3:1 for large text (≥18px bold / ≥24px regular)
    • Non-text elements (icons, form controls) meet 3:1 vs adjacent colors
    • Focus indicators: 3px solid primary-500 outline + 2px white outline offset
    • Active elements in dark mode tested against neutral-900 backgrounds
    • Color is never the sole indicator of state (accompany with icon or text)
    • Semantic colors paired with label text (e.g., "Error: ..." not just red)

    6.2 Keyboard Navigation

    Pattern Implementation
    Tab order Logical DOM order (left→right, top→bottom). No tabindex > 0
    Arrow keys List navigation (dropdown, kanban, table rows)
    Escape Close modals, dropdowns, menus, search
    Enter/Space Activate focused element
    Tab Move to next focusable element
    Shift+Tab Move to previous focusable element
    Home/End First/last item in list, first/last tab panel
    ⌘K Global search activate
    ? Show keyboard shortcuts help overlay

    Tab traps: - Modals: Focus trapped inside modal while open - Dropdown: Focus trapped until Escape or selection - Slide-over panels: Focus trapped while panel is open

    6.3 Screen Reader Support

    Element ARIA Pattern
    Sidebar nav role="navigation" with aria-label="Main navigation"
    Modal role="dialog", aria-modal="true", aria-labelledby="header-id"
    Tab panels role="tablist", role="tab", role="tabpanel", aria-selected, aria-controls
    Dropdown menu role="menu", role="menuitem"
    Alert/toast role="alert" with aria-live="polite" (or "assertive" for errors)
    Progress indicator role="progressbar", aria-valuenow, aria-valuemin, aria-valuemax
    Toggle role="switch", aria-checked
    Tooltip aria-describedby on trigger element
    Table sort aria-sort="ascending" or "descending" on header cell
    Drag-and-drop role="button", aria-grabbed (false/true), keyboard alternative
    Search role="combobox" with aria-expanded
    Notification badge aria-label="3 unread notifications"

    6.4 Focus Management

    • Focus goes to first interactive element when a modal/dialog opens
    • Focus returns to trigger element on modal/dialog close
    • Dynamic content (loading → loaded) does not steal focus unless it's a critical update
    • Route changes: focus heading of new page (h1) via tabindex="-1" + .focus()
    • Toast notifications: include in live region but don't move focus
    • Skeleton loading: aria-busy="true" on container

    6.5 Reduced Motion

    • All animations respect prefers-reduced-motion: reduce
    • Substitute: opacity transitions for translate/scale transitions
    • Skeleton shimmer becomes static placeholder
    • Drag ghost becomes instant snap (no bouncy transition)
    • Notification slide-in becomes instant appear
    • Staggered list fade becomes batch appear

    7.1 Theme Switching

    • Default: Follow prefers-color-scheme OS setting
    • Override: Toggle in Settings → Appearance → Light / Dark / System
    • Persistence: Saved to localStorage, synced across sessions
    • Transition: color-scheme on <html>, CSS custom properties swap, 300ms transition on background-color and color
