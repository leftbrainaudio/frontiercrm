    Design System — Implementation Gaps

    Creative audited the wiki specs (1,732 combined lines in 08_BRAND_GUIDELINES.md and ui-design-system.md) against the actual frontend code at ~/FrontierCRM/frontend/. This page documents every gap found, ordered by priority.

    P0 — Must Fix (brand-breaking)

    1. Typography: Inter font not loaded
    File frontend/src/index.css, frontend/package.json
    Current Uses system font stack (ui-sans-serif, system-ui, sans-serif)
    Spec Inter for headings/body, JetBrains Mono for code
    Fix Add @fontsource/inter + @fontsource/jetbrains-mono to packages. Import weights 400, 600, 700 in src/main.tsx. Update fontFamily.sans in index.css to Inter, -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif

    2. Color tokens use gray instead of slate
    File frontend/src/index.css lines 5-34 (@theme block)
    Current --color-surface: #ffffff; --color-surface-secondary: #f9fafb; --color-border: #e5e7eb; --color-text-primary: #111827; --color-text-secondary: #6b7280; --dark-surface: #0f172a;
    Spec neutral-50: #F8FAFC, neutral-100: #F1F5F9, neutral-200: #E2E8F0, neutral-800: #1E293B, neutral-600: #475569, neutral-950: #0B1120
    Fix Re-map all token hex values to match spec §3 of BRAND_GUIDELINES

    3. Missing secondary- (teal) and accent- (amber) token aliases
    File frontend/src/index.css
    Current Uses Tailwind stock emerald/amber/red/sky for badges and data viz
    Spec --color-secondary-50 through -900 (teal #14B8A6 scale) and --color-accent-50 through -900 (amber #F59E0B scale) — full scales
    Fix Add both scales matching spec §3.2 and §3.3

    4. Dark mode defaults to 'light' instead of OS preference
    File frontend/src/store/theme.ts line 12
    Current theme: (localStorage.getItem('theme') as Theme) || 'light'
    Spec Should respect prefers-color-scheme OS setting, then fall back to 'light'
    Fix Change fallback to window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light'

    5. Logo assets missing from sidebar
    File frontend/src/components/layout/sidebar.tsx lines 43-51
    Current Blue F letter icon + text "FrontierCRM"
    Spec Logo SVGs at public/logos/logo-mark.svg (collapsed) and public/logos/logo.svg (expanded)
    Fix Replace with <img src="/logos/logo-mark.svg"> and full lockup respectively
    Status ⏸ On hold (logo quality not approved)
