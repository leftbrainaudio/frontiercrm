FrontierCRM Architecture

SMB/Mid-Market AI-Native CRM · v1 Architecture · 2026-06-28

Stack Decision Highlights

  • • Backend: Python FastAPI (async + AI/ML pipeline)
  • • Frontend: React 19 + TypeScript + Next.js 15
  • • Mobile: PWA-first (defer RN to Phase 4)
  • • Auth: Clerk (JWT + SSO + MFA)
  • • Hosting: GCP Cloud Run + Cloud SQL + Memorystore

Data Layer Selections

  • • PostgreSQL 16 + TimescaleDB (RLS multi-tenancy)
  • • Redis 7 (cache · pub/sub · Celery broker)
  • • Meilisearch (typo-tolerant full-text search)
  • • Cloudflare R2 (S3-compatible, zero egress)
  • • Custom fields via JSONB, analytics via hypertables

Key Architecture Decisions

  • • REST-first API (webhooks, caching, public API)
  • • Modular monolith (not microservices — 3 engineers)
  • • Row-level security for multi-tenancy
  • • Celery + Redis for background jobs
  • • Resend + SES for email delivery
